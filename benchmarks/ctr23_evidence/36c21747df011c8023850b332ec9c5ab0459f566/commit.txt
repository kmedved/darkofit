Default ordered_boosting=False for ChimeraBoostRegressor

LOO correction reduces gradient signal per tree on large smooth regression
datasets, causing slower convergence and a quality ceiling vs the field.
Disabling it for regression by default yields +3.6pp on Grinsztajn RMSE
with zero cost to classification accuracy or calibration.

ChimeraBoostClassifier keeps ordered_boosting=True (LOO still wins there).
Benchmark harness updated to use class defaults rather than always passing
True, so --no-ordered-boosting now forces False for both tasks.

Also ships: Grinsztajn HuggingFace benchmark suite (59 datasets), calibration
MCB column in make_tables.py, and ChimeraBoostEns10 in the default model set.
